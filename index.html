📱 Cómo subir tu proyecto a Android Studio
Cuando generas un proyecto Android en el Arquitecto, recibes archivos Java, XML, Gradle, etc. Aquí te explico paso a paso cómo meterlos a Android Studio para que funcionen.
⚠️ Importante: Cada proyecto Android tiene archivos diferentes (distintas Activities, layouts, servicios), pero la ESTRUCTURA de carpetas siempre es la misma. Lo que cambia es el contenido, no dónde va.
📂 ESTRUCTURA DE UN PROYECTO ANDROID
Todos los proyectos Android siguen esta estructura. Memorízala y nunca te pierdes:
📁 MiProyecto/
📄 build.gradle ← Gradle del proyecto (general)
📄 settings.gradle
📁 app/
📄 build.gradle ← Gradle de la app (dependencias, SDK)
📁 src/
📁 main/
📄 AndroidManifest.xml ← Permisos y Activities
📁 java/com/tu/paquete/ ← Archivos .java
📄 MainActivity.java
📄 LoginActivity.java
📄 Adaptador.java
📁 res/ ← Recursos (XML layouts, imágenes, strings)
📁 layout/ ← Pantallas XML
📄 activity_main.xml
📄 activity_login.xml
📁 values/ ← Colores, textos, estilos
📄 strings.xml
📄 colors.xml
📁 drawable/ ← Imágenes e íconos
🎯 ¿DÓNDE VA CADA TIPO DE ARCHIVO?
Cuando copies código del Arquitecto, pon cada archivo en su lugar correcto:
J
Archivos .java (Activities, Modelos, Adaptadores)
app → src → main → java → com → tu → paquete → AQUÍ
Cada clase Java va aquí. MainActivity.java, LoginActivity.java, etc. El paquete es como "com.propart.diagnostic"
X
Layouts XML (pantallas)
app → src → main → res → layout → AQUÍ
activity_main.xml, activity_login.xml, item_producto.xml... todo layout aquí.
M
AndroidManifest.xml
app → src → main → AndroidManifest.xml
Aquí se registran Activities, permisos (CAMERA, INTERNET, BLUETOOTH, LOCATION), y el nombre de la app.
G
build.gradle (Module: app)
app → build.gradle
Dependencias (Firebase, ZXing, BLE, etc.), minSdk, targetSdk, applicationId van aquí. NO confundir con el build.gradle de la raíz.
R
values/ (strings.xml, colors.xml, styles.xml)
app → src → main → res → values → AQUÍ
Textos de la app, colores del tema, estilos globales.
D
drawable/ (imágenes, íconos SVG)
app → src → main → res → drawable → AQUÍ
Logos, íconos, backgrounds personalizados. Formatos: PNG, XML shape, SVG (vector).
🛠️ ELIGE TU MÉTODO
✅ Mejor para: Cuando te dieron archivos sueltos (de Claude, de un tutorial, o de aquí del Arquitecto) y necesitas armarlos en un proyecto nuevo.
1
Abre Android Studio → New Project
Abre Android Studio. En la pantalla de inicio toca "New Project".
Android Studio → New Project
2
Elige "Empty Activity"
En las plantillas, selecciona "Empty Views Activity" (no Compose, a menos que tu código sea Compose). Esto te da la estructura básica lista.
💡 Si tu proyecto usa Fragments, igual empieza con Empty Activity. Los Fragments se agregan después.
3
Configura nombre y paquete
Name: El nombre de tu app (ej: "ProPart Diagnostic")
Package name: Identificador único (ej: com.propart.diagnostic)
Language: Java
Minimum SDK: API 24 (Android 7.0) recomendado
⚠️ El Package name DEBE coincidir con el que viene en los archivos Java. Si tu código dice package com.propart.diagnostic; entonces aquí pones exactamente eso.
4
Dale "Finish" — espera que Gradle sincronice
Android Studio crea el proyecto y baja dependencias. La barra de abajo dice "Gradle sync". Espera a que termine (puede tardar 1-3 minutos la primera vez).
💡 Si falla el Gradle, revisa que tengas internet y que tu SDK esté actualizado: Tools → SDK Manager.
5
Cambia a vista "Project" (no "Android")
En el panel izquierdo, arriba dice "Android". Cámbialo a "Project" para ver la estructura REAL de carpetas. Así encuentras exactamente dónde poner cada archivo.
Panel izquierdo → Dropdown "Android" → Cambiar a "Project"
6
Reemplaza/agrega los archivos Java
Navega a app/src/main/java/com/tu/paquete/

Para MainActivity.java: Ya existe uno que creó Android Studio. Ábrelo y REEMPLAZA todo el contenido con el código que te dio el Arquitecto. Copia y pega completo.

Para archivos nuevos (LoginActivity.java, Adapter, etc.): Click derecho en la carpeta del paquete → New → Java Class → Pon el nombre exacto → Pega el código completo.
app → src → main → java → com.tu.paquete → Click derecho → New → Java Class
7
Reemplaza/agrega los layouts XML
Navega a app/src/main/res/layout/

activity_main.xml ya existe — ábrelo y pega el código XML nuevo.
Nuevos layouts: Click derecho en layout/ → New → Layout Resource File → Nombre exacto (ej: activity_login) → Pega el XML.
app → src → main → res → layout → Click derecho → New → Layout Resource File
8
Actualiza el AndroidManifest.xml
Abre app/src/main/AndroidManifest.xml. Agrega:

• Permisos que necesita tu app (INTERNET, CAMERA, BLUETOOTH, etc.) — van ANTES de la etiqueta <application>
• Activities nuevas — cada Activity que agregas debe registrarse con una etiqueta <activity> dentro de <application>
⚠️ Si no registras una Activity en el Manifest, la app TRUENA al intentar abrirla. Error clásico: "Activity not found".
9
Actualiza build.gradle (Module: app)
Abre app/build.gradle (el que dice "Module: app", NO el de "Project").

Agrega las dependencias que necesita tu proyecto en el bloque dependencies { }. Ejemplo:
dependencies {
  implementation 'com.google.firebase:firebase-bom:32.0.0'
  implementation 'com.journeyapps:zxing-android-embedded:4.3.0'
  // ... las que indique tu proyecto
}
💡 Después de editar build.gradle, Android Studio te pide "Sync Now" arriba — DALE. Si no sale, ve a File → Sync Project with Gradle Files.
10
Conecta tu celular y dale ▶ Run
Conecta tu Android por USB. Activa Depuración USB en opciones de desarrollador.
Arriba en Android Studio aparece tu dispositivo. Dale al botón verde ▶ (Run).
Si hay errores, léelos abajo en "Build" — casi siempre es un import faltante o un recurso que no encuentra.
Toolbar → Seleccionar dispositivo → ▶ Run 'app'
✅ Mejor para: Cuando el proyecto ya está en un repo de GitHub (como tus proyectos en github.com/84-fmill-mel30) y quieres bajarlo completo ya armado.
1
Abre Android Studio → "Get from VCS"
En la pantalla de inicio, en lugar de "New Project", busca el botón "Get from Version Control" (o "Get from VCS").
Android Studio → Get from VCS
2
Pega la URL del repositorio
En "URL" pega la dirección de tu repo. Ejemplo:
https://github.com/84-fmill-mel30/tu-repo.git
En "Directory" elige dónde quieres guardar el proyecto en tu computadora.
Dale "Clone".
3
Android Studio detecta el proyecto
Si el repo tiene un proyecto Android válido (con build.gradle), Android Studio lo reconoce automáticamente y empieza a sincronizar Gradle. Espera a que termine.
⚠️ Si el repo solo tiene archivos sueltos (sin estructura de proyecto), Android Studio NO lo reconoce. En ese caso usa el Método Manual y copia los archivos uno por uno.
4
Verifica SDK y dependencias
Si el proyecto fue hecho con una versión diferente de Android SDK, puede pedir que bajes componentes extra. Android Studio te avisa con links azules — dale click para instalar lo que falta.
💡 Revisa que el compileSdkVersion y buildToolsVersion en build.gradle coincidan con lo que tienes instalado. Si no, cámbialos a tu versión.
5
Run ▶ y prueba
Conecta tu dispositivo, dale ▶ Run. Si todo el repo estaba bien armado, debería compilar directo.
🔄 ALTERNATIVA: ABRIR PROYECTO EXISTENTE
Si ya bajaste el repo con git clone desde la terminal, o lo descargaste como ZIP:
A
Android Studio → File → Open → Navega a la carpeta del proyecto → Selecciona la carpeta raíz (la que tiene el build.gradle) → Open
File → Open → Seleccionar carpeta raíz → OK
🔥 ERRORES COMUNES Y SOLUCIONES
!
"Cannot resolve symbol R"
Causa: Error en un XML (layout, strings, colors). Un solo carácter mal rompe la R.
Solución: Build → Clean Project, luego Build → Rebuild. Si sigue, revisa tus XMLs buscando errores rojos.
!
"Activity not found" / App se cierra
Causa: No registraste la Activity en AndroidManifest.xml
Solución: Agrega <activity android:name=".NombreActivity"/> dentro de <application>
!
"Gradle sync failed"
Causa: Dependencia mal escrita, sin internet, o versión incompatible.
Solución: Lee el error abajo. Generalmente dice qué librería falla. Verifica la versión en el build.gradle.
!
"Package name does not match"
Causa: El paquete en el Java dice com.x.y pero la carpeta o el Manifest dice otro.
Solución: Los 3 deben coincidir: la carpeta, la línea package en Java, y el package en AndroidManifest.
!
"No devices found" / No detecta el celular
Causa: USB solo carga, o no activaste depuración USB.
Solución: En tu cel: Ajustes → Acerca del teléfono → Toca "Número de compilación" 7 veces → Regresa → Opciones de desarrollador → Activa "Depuración USB". Cambia el cable a "Transferir archivos".
